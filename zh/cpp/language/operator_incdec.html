自增/自减运算符增加或减小对象的值。

运算符名 语法 可重载 原型示例（对于类 class T）
类内定义 类外定义
前自增 ++a 是 T& T::operator++(); T& operator++(T& a);
前自减 --a 是 T& T::operator--(); T& operator--(T& a);
后自增 a++ 是 T T::operator++(int); T operator++(T& a, int);
后自减 a-- 是 T T::operator--(int); T operator--(T& a, int);
注解
  • 内建运算符的前置版本返回引用而后置版本返回值，典型的用户定义重载也遵循该模式，从而使用户定义运算符可以与内建版本相同的方式使用。然而，用户定义重载中，能以任何类型为返回类型（包括 void）。
  • int 形参是用于区别运算符的前置和后置版本的空设形参。调用用户定义的运算符时，该形参中传递的值始终为零，但通过函数调用记法调用这个运算符时可以更改它（例如 a.operator++(2) 或 operator++(a, 2)）。

解释

前自增 与 前自减 运算符增加或减小对象的值，并返回到其结果的引用。

后自增 与 后自减 创建对象的副本，增加或减小对象的值，并返回自增或自减前的副本。

内建的前置运算符

前置自增与自减表达式的形式为

++ 表达式
-- 表达式
1) 前置自增（前自增）
2) 前置自减（前自减）

内建前置自增或自减运算符的操作数 表达式 必须是非布尔的算术类型或指向完整对象类型的指针类型的可修改（非 const）左值。对于非布尔操作数，表达式 ++x 严格等价于 x += 1，而表达式 --x 严格等价于 x -= 1，即前置自增与自减是标识修改后的操作数的左值表达式。所有针对算术运算符定义的算术转换规则和指针算术规则都适用，并由此确定应用到操作数的隐式转换（若存在）以及表达式的返回类型。

若前自增运算符的运算数为 bool 类型，则它被设为 true (弃用)。 (C++17 前)

在针对用户定义运算符的重载决议中，对于每个 bool 以外的可选地有 volatile 限定的算术类型 A，和对于每个可选地有 volatile 限定的指向可选地有 cv 限定的对象类型的指针 P，下列函数签名参与重载决议：

A& operator++(A&)
bool& operator++(bool&)
 (弃用)(C++17 前)
P& operator++(P&)
A& operator--(A&)
P& operator--(P&)

内建的后置运算符

后置自增与自减表达式拥有形式

表达式 ++
表达式 --
1) 后置自增（后自增）
2) 后置自减（后自减）

内建后置自增或自减运算符的操作数 表达式 必须是非布尔的算术类型或指向完整对象类型的指针类型的可修改（非 const）左值。结果是操作数原值的纯右值副本。作为副作用，对于非布尔操作数，表达式 x++ 如同 x += 1 一般修改其操作数的值，而表达式 x-- 如同 x -= 1 一般修改其操作数的值。所有为算术运算符定义的算术转换规则和指针算术规则都适用，并由此确定应用到操作数的隐式转换（若存在）以及表达式的返回类型。

若后自增运算符的操作数为 bool 类型，则它被设为 true (弃用)。 (C++17 前)

在针对用户定义运算符的重载决议中，对于每个 bool 以外的可选地有 volatile 限定的算术类型 A，和对于每个可选地有 volatile 限定的指向可选地有 cv 限定的对象类型的指针 P，下列函数签名参与重载决议：

A operator++(A&, int)
bool operator++(bool&, int)
 (弃用)(C++17 前)
P operator++(P&, int)
A operator--(A&, int)
P operator--(P&, int)

示例

运行此代码
#include <iostream>
 
int main()
{
    int n1 = 1;
    int n2 = ++n1;
    int n3 = ++ ++n1;
    int n4 = n1++;
//  int n5 = n1++ ++;   // 错误
//  int n6 = n1 + ++n1; // 未定义行为
    std::cout << "n1 = " << n1 << '\n'
              << "n2 = " << n2 << '\n'
              << "n3 = " << n3 << '\n'
              << "n4 = " << n4 << '\n';
}

输出： 

n1 = 5
n2 = 2
n3 = 4
n4 = 4

注解

因为涉及到了副作用，所以必须谨慎使用内建的自增和自减运算符，以避免违背定序规则所致的未定义行为。

由于后自增和后自减期间会构造对象的临时副本，故在不使用返回值的语境中前自增或前自减运算符通常更有效率。

标准库

自增和自减运算符对许多标准库类型重载。特别是每个旧式迭代器 (LegacyIterator) 都重载了 operator++，而且每个旧式双向迭代器 (LegacyBidirectionalIterator) 都重载了 operator--，即便这些运算符对于某些特定迭代器是无操作的。
